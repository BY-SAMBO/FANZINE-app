Limpieza

1-2 dias — Prioridad maxima
Urgente

Lo primero es limpiar lo que puede causar problemas inmediatos. Como cuando en la cocina primero se limpia la plancha antes de cocinar. Estos cambios son rapidos y reducen el riesgo de errores en el dia a dia.

  • Eliminar el archivo duplicado pos-live-store.ts
    Borrar el archivo que no se usa para evitar confusion. Toma 5 minutos.
  • Corregir asignacion de toppings al producto incorrecto
    Anclar la seleccion de toppings al producto especifico usando un ID unico, no la posicion en la lista.
  • Agregar confirmacion antes de limpiar pedido
    Mostrar un dialogo "Esta seguro que desea borrar el pedido?" antes de limpiar. Previene perdida accidental.
  • Crear migraciones para tablas faltantes
    Documentar las tablas pos_modifier_cache y pos_sales_log en migraciones para que se puedan recrear automaticamente.
1

Fundacion

3-5 dias — Base solida para el futuro

Construir una base solida sobre la que se pueda mejorar. Es como reforzar los cimientos del restaurante antes de remodelar. Estos cambios eliminan la duplicacion de codigo y establecen un flujo de pedido mas claro.

  • Extraer logica compartida de stores
    Crear un "store base" que tenga toda la logica comun de pedidos (agregar/quitar productos, toppings, cobro). Caja y delivery lo usaran sin duplicar codigo.
  • Crear componente compartido de toppings
    Un solo componente de toppings que se use en las 3 pantallas (caja, cliente, delivery). Cambios en uno se reflejan en todos.
  • Agregar maquina de estados para el pedido
    Definir estados claros: vacio → armando → toppings → cobro → enviado → impreso. El sistema no permitira saltar pasos.
  • Agregar manejo de errores robusto
    Cada paso del proceso de venta debe tener un manejo claro de errores: que pasa si falla, como se recupera, y que se le muestra al cajero.
  • Agregar validacion antes de enviar a Fudo
    Verificar que todos los productos tengan ID de Fudo, que los precios sean validos, y que el metodo de pago sea correcto antes de intentar la venta.
2

Comunicacion

2-3 dias — Pantallas que no se pierden

Mejorar la comunicacion entre las pantallas del cajero y del cliente. Actualmente, si se pierde la conexion, las pantallas se desincronizan. Despues de esta fase, se reconectaran solas.

  • Agregar reconexion automatica
    Si la conexion se cae, el sistema debe intentar reconectarse automaticamente cada 3 segundos. Cuando se reconecte, sincronizar el estado completo.
  • Agregar cola de mensajes
    Si un mensaje no se puede enviar, guardarlo y enviarlo cuando vuelva la conexion. Los mensajes no se pierden.
  • Mostrar indicador de desconexion
    Cuando las pantallas pierdan contacto, mostrar un banner visible: "Conexion perdida - Reconectando..." para que el cajero sepa.
  • Optimizar tamano de mensajes
    Enviar solo los cambios (deltas) en vez del pedido completo. Reduce el trafico y la latencia.
3

UX / UI

3-5 dias — Facil de usar, dificil de equivocarse

Hacer que el sistema sea mas facil de usar y mas dificil de cometer errores. Como redisenar la cocina para que los utensilios mas usados esten mas cerca.

  • Layout responsive
    Hacer que las pantallas funcionen en monitores grandes, tablets y celulares. El layout se adapta al tamano de la pantalla.
  • Mejor retroalimentacion al usuario
    Animaciones al agregar productos, indicadores de carga al procesar, mensajes de exito y error claros y visibles.
  • Mejoras de accesibilidad
    Mejor contraste, botones mas grandes (minimo 44x44px), etiquetas para lectores de pantalla, navegacion por teclado.
  • Desactivar controles cuando no aplican
    Metodo de pago deshabilitado si no hay productos. Boton de cobrar deshabilitado si el pedido esta vacio. Prevenir acciones que no tienen sentido.
  • Soporte movil para casos de emergencia
    Permitir tomar un pedido desde el celular si la computadora principal falla. No sera la experiencia principal, pero funcionara.
4

Robustez

2-3 dias — Que nada se pierda

Asegurar que el sistema sea resistente a fallos. Que si algo sale mal, se recupere solo y no se pierda informacion. Como instalar un generador de respaldo en el restaurante.

  • Cola de impresion con persistencia
    Si la impresora falla, los tickets se guardan y se reimprimen automaticamente cuando la impresora vuelva. Nunca se pierde un ticket.
  • Claves de idempotencia
    Si el sistema envia la misma venta dos veces a Fudo (por un error de red), Fudo la rechaza como duplicada en vez de crearla doble.
  • Validacion pre-vuelo
    Antes de enviar a Fudo, verificar que la conexion funcione, que los productos existan, y que los precios sean correctos. Avisar al cajero si algo no cuadra.
  • Auto-sincronizacion del cache de toppings
    Refrescar automaticamente los toppings disponibles cada dia, o cuando se detecten cambios en Fudo. Sin intervencion manual.

Siguientes pasos

Para comenzar la reconstruccion, se recomienda:

  1. Iniciar la Fase 0 inmediatamente. Los 4 items de limpieza se pueden completar en 1-2 dias y eliminan los riesgos mas criticos.
  2. Planificar la Fase 1 para la siguiente semana. Requiere mas tiempo pero establece la base para todas las mejoras futuras.
  3. Evaluar prioridades entre Fases 2-4. Dependiendo de las necesidades del restaurante, se puede ajustar el orden. Si las desconexiones son frecuentes, priorizar la Fase 2. Si la interfaz causa mas problemas, priorizar la Fase 3.

Cada fase es independiente una vez completada la Fase 1. Se puede avanzar en el orden que tenga mas sentido para el negocio.
